# Getting Started

This guide will walk you through setting up the Hyperbridge SDK and creating your first cross-chain connection.

## Prerequisites

Before you begin, make sure you have:

- Node.js 22+ installed
- Basic knowledge of TypeScript/JavaScript
- RPC endpoints for your source and destination chains


## Installation

Install the SDK using your preferred package manager:

<Tabs items={['pnpm', 'yarn', 'npm']}>
<Tab value="npm">
```bash lineNumbers
npm install @hyperbridge/sdk
```
</Tab>
<Tab value="yarn">
```bash lineNumbers
yarn add @hyperbridge/sdk
```
</Tab>
<Tab value="pnpm">
```bash lineNumbers
pnpm add @hyperbridge/sdk
```
</Tab>
</Tabs>

**Using Vite?** If you're using Vite, you'll need to add the Hyperbridge Vite plugin to handle WebAssembly dependencies. See the [Vite Integration guide](/developers/sdk/vite-integration) for setup instructions.

## Quick Setup

### 1. Create Chain Instances

**EVM Chain:**

```typescript lineNumbers title="client.ts" icon=typescript
import { EvmChain } from "@hyperbridge/sdk"

// EvmChain.create() auto-detects the chain ID and resolves the IsmpHost address
const sourceChain = await EvmChain.create("https://data-seed-prebsc-1-s1.binance.org:8545")
```

**Substrate Chain:**

```typescript lineNumbers title="client.ts" icon=typescript
import { SubstrateChain } from "@hyperbridge/sdk"

const hyperbridge = await SubstrateChain.connect({
  stateMachineId: "KUSAMA-4009",
  wsUrl: "wss://gargantua.polytope.technology",
  hasher: "Keccak",
  consensusStateId: "PAS0"
})
```

**Polkadot Hub chain (EVM on Substrate / Revive):**

Use `PolkadotHubChain` when the execution environment is the EVM hosted on a Substrate chain (Revive). It wraps an internal [`EvmChain`](/developers/sdk/api/evm-chain) for RPC reads, contract calls, and transaction helpers, and uses a **Substrate JSON-RPC** endpoint to build storage proofs (`queryProof`, `queryStateProof`) from the main trie and Revive child tries—the same layout the relayer uses for Substrate-EVM.

`PolkadotHubChain.create(evmRpcUrl, substrateRpcUrl, bundlerUrl?)` mirrors [`EvmChain.create`](/developers/sdk/api/evm-chain): it resolves chain id and `IsmpHost` from known deployments, attaches your Substrate endpoint for proofs, and optionally forwards an ERC-4337 **bundler URL** as the third argument (same semantics as `EvmChain.create`).

```typescript lineNumbers title="client.ts" icon=typescript
import { PolkadotHubChain } from "@hyperbridge/sdk"

const evmRpc = "https://your-evm-endpoint.example"
const substrateRpc = "https://your-substrate-endpoint.example"

// Without a bundler (two arguments)
const polkadotHub = await PolkadotHubChain.create(evmRpc, substrateRpc)

// With an ERC-4337 bundler (optional third argument, same as EvmChain.create)
const polkadotHubWithBundler = await PolkadotHubChain.create(
  evmRpc,
  substrateRpc,
  "https://your-bundler.example",
)
```

Or pass full parameters when you already know chain id and host:

```typescript lineNumbers title="client.ts" icon=typescript
import { PolkadotHubChain } from "@hyperbridge/sdk"

const polkadotHub = PolkadotHubChain.fromParams({
  chainId: 420420417,
  rpcUrl: "https://your-evm-endpoint.example",
  host: "0xYourIsmpHostAddress",
  substrateRpcUrl: "https://your-substrate-endpoint.example",
  // Optional — defaults exist for some known chain IDs (same as EvmChain)
  consensusStateId: "PAS0",
  // Optional — ERC-4337 bundler, same as EvmChain
  // bundlerUrl: "https://...",
})
```

| Field | Description |
|-------|-------------|
| `chainId`, `rpcUrl`, `host` | Same as [`EvmChain`](/developers/sdk/api/evm-chain): EVM chain id, JSON-RPC URL, and `IsmpHost` address. |
| `substrateRpcUrl` | Substrate node RPC (HTTP or WebSocket). WebSocket URLs are converted to HTTP for proof RPCs. |
| `consensusStateId` | Optional Hyperbridge consensus state id for this chain (mirrors `EvmChain`). |
| `bundlerUrl` | Optional account-abstraction bundler (mirrors `EvmChain`). |

The instance’s `config` includes every `IEvmConfig` field plus `substrateRpcUrl` (`IPolkadotHubConfig`).

**Proofs:** `queryProof` **must** be called with an explicit finalized (or chosen) block height as `at`. Other `IChain` methods delegate to the inner EVM client, same pattern as other chain types.

**Pharos chain:**

Use `PharosChain` when connecting to a Pharos network. Pharos exposes a standard EVM JSON-RPC surface, but its `eth_getProof` endpoint returns proofs in a custom hexary hash tree format (SHA-256) instead of Ethereum's Merkle-Patricia Trie (Keccak-256). `PharosChain` wraps an internal [`EvmChain`](/developers/sdk/api/evm-chain) for all standard interactions and overrides `queryProof` / `queryStateProof` to handle the Pharos proof format.

`PharosChain.create(rpcUrl, bundlerUrl?)` mirrors [`EvmChain.create`](/developers/sdk/api/evm-chain): it resolves chain id and `IsmpHost` from known deployments, and optionally forwards an ERC-4337 **bundler URL** as the second argument.

```typescript lineNumbers title="client.ts" icon=typescript
import { PharosChain } from "@hyperbridge/sdk"

// Auto-detect chain ID and IsmpHost from the RPC
const pharos = await PharosChain.create("https://rpc.pharos.example")

// With an ERC-4337 bundler (optional second argument, same as EvmChain.create)
const pharosWithBundler = await PharosChain.create(
  "https://rpc.pharos.example",
  "https://your-bundler.example",
)
```

Or pass full parameters when you already know chain id and host:

```typescript lineNumbers title="client.ts" icon=typescript
import { PharosChain } from "@hyperbridge/sdk"

const pharos = PharosChain.fromParams({
  chainId: 688689,
  rpcUrl: "https://rpc.pharos-atlantic.example",
  host: "0xYourIsmpHostAddress",
  // Optional — defaults exist for some known chain IDs (same as EvmChain)
  // consensusStateId: "PHAR",
  // Optional — ERC-4337 bundler, same as EvmChain
  // bundlerUrl: "https://...",
})
```

| Field | Description |
|-------|-------------|
| `chainId`, `rpcUrl`, `host` | Same as [`EvmChain`](/developers/sdk/api/evm-chain): EVM chain id, JSON-RPC URL, and `IsmpHost` address. |
| `consensusStateId` | Optional Hyperbridge consensus state id for this chain (mirrors `EvmChain`). |
| `bundlerUrl` | Optional account-abstraction bundler (mirrors `EvmChain`). |

**Proofs:** `queryProof` **must** be called with an explicit finalized block height as `at`. The returned proof is a SCALE-encoded `PharosStateProof` containing hexary hash tree proof nodes, storage values, non-existence proofs, and account proofs. Other `IChain` methods delegate to the inner EVM client.

| Chain | Chain ID |
|-------|----------|
| Pharos Mainnet | 688600 |
| Pharos Atlantic (Testnet) | 688689 |

For detailed configuration options and supported chains, see:
- [EvmChain API Reference](/developers/sdk/api/evm-chain) (shared EVM surface for Polkadot Hub and Pharos)
- [SubstrateChain API Reference](/developers/sdk/api/substrate-chain)

### 2. Create Query Client

```typescript lineNumbers title="client.ts" icon=typescript
import { createQueryClient } from "@hyperbridge/sdk"

const queryClient = createQueryClient({
  url: "https://gargantua.indexer.polytope.technology"
})
```

Polytope Labs provides a free indexer service for both mainnet and testnet. The endpoints are actually GraphQL APIs, so a GraphiQL playground is available at the endpoints as well.

| Network | Indexer URL |
|---------|-------------|
| Mainnet | https://nexus.indexer.polytope.technology |
| Testnet | https://gargantua.indexer.polytope.technology |

---

### 3. Initialize IsmpClient

```typescript lineNumbers title="client.ts" icon=typescript
import { IsmpClient } from "@hyperbridge/sdk"

const indexer = new IsmpClient({
  queryClient,
  pollInterval: 1_000, // Poll every 1 second
  source: sourceChain,
  dest: destChain,
  hyperbridge: hyperbridge
})
```

**Configuration Options:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `queryClient` | `QueryClient` | Query client for the Hyperbridge indexer |
| `pollInterval` | `number` | Polling interval in milliseconds (default: 1000) |
| `source` | `IChain` | Source chain instance (`EvmChain`, `SubstrateChain`, `PolkadotHubChain`, `PharosChain`, …) |
| `dest` | `IChain` | Destination chain instance |
| `hyperbridge` | `IChain` | Hyperbridge chain instance (often `SubstrateChain`) |

For complete API documentation and usage examples, see the [IsmpClient API Reference](/developers/sdk/api/ismp-client).

## Complete Example

```typescript lineNumbers title="client.ts" icon=typescript
import { 
  IsmpClient, 
  createQueryClient, 
  EvmChain,
  SubstrateChain 
} from "@hyperbridge/sdk"

async function main() {
  // 1. Create query client
  const queryClient = createQueryClient({
    url: "https://gargantua.indexer.polytope.technology"
  })

  // 2. Create chain instances — auto-detects chain ID and IsmpHost address
  const sourceChain = await EvmChain.create("https://data-seed-prebsc-1-s1.binance.org:8545")
  const destChain   = await EvmChain.create("https://rpc.chiadochain.net")

  const hyperbridgeChain = await SubstrateChain.connect({
    stateMachineId: "KUSAMA-4009",
    wsUrl: "wss://gargantua.polytope.technology",
    hasher: "Keccak",
    consensusStateId: "PAS0"
  })

  // 3. Initialize IsmpClient
  const indexer = new IsmpClient({
    queryClient,
    pollInterval: 1_000,
    source: sourceChain,
    dest: destChain,
    hyperbridge: hyperbridgeChain
  })

  console.log("✓ SDK initialized successfully!")

  // 4. Track a request (example)
  const requestHash = "0xYourRequestHashHere"
  
  for await (const status of indexer.postRequestStatusStream(requestHash)) {
    console.log(`Status: ${status.status}`)
    
    if (status.status === "DESTINATION" || status.status === "PENDING_TIMEOUT") {
      break
    }
  }
  
  // Clean up
  await hyperbridgeChain.disconnect()
}

main().catch(console.error)
```

# Running a Prover

The Hyperbridge BEEFY consensus prover generates cryptographic proofs that attest to the finality of cross-chain messages on the Hyperbridge parachain. For an overview of how provers work, their incentive model, and the path to becoming a Collator, see [Explore: Provers](/developers/explore/provers).

## Running with Docker

The prover is available at the official Docker repository [`polytopelabs/prover`](https://hub.docker.com/r/polytopelabs/prover).

```bash lineNumbers
docker pull polytopelabs/prover:latest
```

### Configuration

Create a `config.toml` file. Use the **Testnet** tab for proving Gargantua finality on Paseo, or **Mainnet** for proving Nexus finality on Polkadot.

<Tabs items={['Testnet', 'Mainnet']}>
  <Tab value="Testnet">

```toml lineNumbers
[prover]
# Polkadot Paseo (testnet relay chain) RPC websocket endpoint
relay_rpc_ws = "wss://paseo.ibp.network"
# Gargantua (testnet parachain) RPC websocket endpoint
para_rpc_ws = "wss://gargantua.rpc.polytope.technology"
# Gargantua's para_id on Paseo
para_ids = [4009]
# Proof strategy: "ecdsa" (no GPU, testnet default) or "sp1" (requires CUDA GPU)
proof_variant = "ecdsa"

[substrate]
# Gargantua parachain RPC
rpc_ws = "wss://gargantua.rpc.polytope.technology"
# SR25519 secret key for signing proof submissions (hex-encoded)
signer = "0x<your_sr25519_secret_key>"

[beefy]
# Consensus state identifier used by pallet-ismp
consensus_state_id = "PAS0"
# Minimum number of blocks after a message before proving finality (0 = prove immediately)
minimum_finalization_height = 0
```

  </Tab>
  <Tab value="Mainnet">

```toml lineNumbers
[prover]
# Polkadot (mainnet relay chain) RPC websocket endpoint
relay_rpc_ws = "wss://polkadot.ibp.network"
# Nexus (mainnet parachain) RPC websocket endpoint
para_rpc_ws = "wss://nexus.rpc.polytope.technology"
# Nexus's para_id on Polkadot
para_ids = [3367]
# Proof strategy: "sp1" (mainnet — requires CUDA GPU)
proof_variant = "sp1"

[substrate]
# Nexus parachain RPC
rpc_ws = "wss://nexus.rpc.polytope.technology"
# SR25519 secret key for signing proof submissions (hex-encoded)
signer = "0x<your_sr25519_secret_key>"

[beefy]
# Consensus state identifier used by pallet-ismp
consensus_state_id = "DOT0"
# Minimum number of blocks after a message before proving finality (0 = prove immediately)
minimum_finalization_height = 0
```

  </Tab>
</Tabs>

<Callout type="warn">
The `signer` field is the SR25519 secret key that will sign proof submissions. Proof submission is a **signed extrinsic**, so the account must hold a small balance for transaction fees. Fees are **refunded when a proof is accepted**, but the submitter is **charged when a proof arrives too late** (i.e. the chain has already advanced past it). Accepted proofs also earn `$BRIDGE` rewards and Reputation Asset tokens.
</Callout>

### Host Prerequisites (SP1 / GPU mode)

Before running with `proof_variant = "sp1"`, the host needs the following. CUDA libraries themselves ship inside the prover image — no host CUDA install required.

<Steps>
<Step>
### Install the NVIDIA driver

Install a driver that matches your GPU (e.g. ≥ 580 for Blackwell RTX 50xx). Verify:

```bash lineNumbers
nvidia-smi
```
</Step>
<Step>
### Install Docker and Docker Compose v2

Follow the official Docker install guide for your distro:
[Install Docker Engine](https://docs.docker.com/engine/install/).
The `docker compose` plugin (v2) is included with current Docker Engine releases — the legacy `docker-compose` binary is not supported.
</Step>
<Step>
### Install the NVIDIA Container Toolkit

Install per the official guide so Docker can expose the GPU to containers:
[Installing the NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html),
then configure the Docker runtime:
[Configuring Docker](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html#configuring-docker).

Verify the toolkit can hand the GPU to a container:

```bash lineNumbers
docker run --rm --gpus all nvidia/cuda:12.8.0-base-ubuntu24.04 nvidia-smi
```
</Step>
</Steps>

### Docker Compose

Create a `docker-compose.yml`. The **Mainnet** variant adds a GPU reservation and a bind mount to `$HOME/.sp1` so the ~8 GB of SP1 circuit artifacts downloaded on first run survive container recreates.

<Tabs items={['Testnet', 'Mainnet']}>
  <Tab value="Testnet">

```yaml lineNumbers
services:
  tesseract-prover:
    image: polytopelabs/prover:latest
    restart: always
    network_mode: host
    volumes:
      - ./config.toml:/app/config.toml:ro
    environment:
      RUST_LOG: info,moongate_core=off,sp1_beefy=trace,beefy_prover=trace
    command: ["--config", "/app/config.toml"]
```

  </Tab>
  <Tab value="Mainnet">

```yaml lineNumbers
services:
  tesseract-prover:
    image: polytopelabs/prover:latest
    restart: always
    network_mode: host
    volumes:
      - ./config.toml:/app/config.toml:ro
      # Persist the ~8 GB of SP1 circuit artifacts on the host so they survive
      # container recreates. The directory will be created on first run.
      - ${HOME}/.sp1:/root/.sp1
    environment:
      RUST_LOG: info,moongate_core=off,sp1_beefy=trace,beefy_prover=trace
    command: ["--config", "/app/config.toml"]
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```

  </Tab>
</Tabs>

Start the prover:

```bash lineNumbers
docker compose up -d
```

View logs:

```bash lineNumbers
docker compose logs -f tesseract-prover
```

### System Requirements

#### ECDSA proofs (testnets)

| Resource | Minimum |
|----------|---------|
| CPU | 4 cores |
| RAM | 8 GB |
| Network | 50 Mb/s |
| GPU | Not required |

#### SP1 proofs (mainnet)

| Resource | Minimum |
|----------|---------|
| CPU | 32 cores |
| RAM | 64 GB |
| Network | 100 Mb/s |
| GPU | NVIDIA RTX 4090 or newer (CUDA required) |

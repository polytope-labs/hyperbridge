# HyperFungibleToken

Deploy this on every **remote chain** where the token doesn't have native supply. The contract inherits from ERC20 and [HyperApp](/developers/evm/api/hyperapp), so it is both the token and the bridge in one contract.

When tokens arrive from the home chain, the contract mints new supply to the recipient. When a user bridges back, it burns their tokens and dispatches an ISMP POST request to the home chain. On timeout, tokens are re-minted to the original sender. The HFT has no initial supply — tokens are only minted when cross-chain transfers arrive.

## Deployment

<Steps>
<Step>
### Deploy with CREATE2

Use CREATE2 for deterministic addresses across all chains.

```solidity lineNumbers
import {HyperFungibleToken} from "@hyperbridge/core/apps/HyperFungibleToken.sol";
import {IHyperFungibleToken} from "@hyperbridge/core/interfaces/IHyperFungibleToken.sol";

bytes32 salt = keccak256("my-token-v1");
HyperFungibleToken token = new HyperFungibleToken{salt: salt}("Wrapped USDC", "wUSDC", msg.sender);
```
</Step>
<Step>
### Configure

```solidity lineNumbers
IHyperFungibleToken(address(token)).configure(
    IHyperFungibleToken.ConfigOptions({
        host: ISMP_HOST_ADDRESS,
        dispatcher: CALL_DISPATCHER_ADDRESS
    })
);
```

Find the `IsmpHost` address for your chain on the [contract addresses](/developers/evm/contract-addresses/mainnet) page.
</Step>
<Step>
### Register peer chains

Register the WrappedHFT on the home chain and any other HFT peers:

```solidity lineNumbers
import {StateMachine} from "@hyperbridge/core/libraries/StateMachine.sol";

// Register the WrappedHFT on Ethereum (home chain)
IHyperFungibleToken(address(token)).addChain(
    StateMachine.evm(1),
    abi.encodePacked(ethereumWrapperAddress)
);

// Register the HFT peer on Base
IHyperFungibleToken(address(token)).addChain(
    StateMachine.evm(8453),
    abi.encodePacked(baseHftAddress)
);
```

On each peer chain, do the reverse — register this chain's HFT address.
</Step>
</Steps>

## Sending Tokens

Users call `send()` to burn tokens and dispatch a cross-chain transfer:

```solidity lineNumbers
import {StateMachine} from "@hyperbridge/core/libraries/StateMachine.sol";
import {IHyperFungibleToken} from "@hyperbridge/core/interfaces/IHyperFungibleToken.sol";

// Pay fees in native token
IHyperFungibleToken(tokenAddress).send{value: nativeFee}(
    IHyperFungibleToken.SendParams({
        dest: StateMachine.evm(42161),
        to: abi.encodePacked(recipientAddress),
        amount: 1 ether,
        timeout: 3600,
        relayerFee: relayerFee,
        data: ""
    })
);
```

To pay fees in the host's fee token instead, send with `msg.value = 0`. The contract pulls fee tokens from the caller via `transferFrom`:

```solidity lineNumbers
IERC20(feeToken).approve(tokenAddress, feeAmount);

IHyperFungibleToken(tokenAddress).send(
    IHyperFungibleToken.SendParams({
        dest: StateMachine.evm(42161),
        to: abi.encodePacked(recipientAddress),
        amount: 1 ether,
        timeout: 3600,
        relayerFee: relayerFee,
        data: ""
    })
);
```

### Fee Estimation

Use the inherited [HyperApp](/developers/evm/api/hyperapp) `quote()` and `quoteNative()` methods to estimate fees on-chain. For a higher-level experience, use the [TypeScript SDK](/developers/sdk/hyper-fungible-token) which handles gas estimation, cross-chain fee token conversion, and decimal scaling automatically.

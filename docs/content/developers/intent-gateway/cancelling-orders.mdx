# Cancelling Orders

When a user wants to reclaim their escrowed tokens — because no solver filled the order, or the order expired — they call `cancelOrder` on the `IntentGatewayV2` contract. The contract checks that the order has not already been settled, then routes to the appropriate refund path depending on whether the order is same-chain or cross-chain, and which chain the call originates from.

The `IntentGateway.cancelOrder()` SDK method abstracts over all three paths. You pass the order, an `IndexerClient` for proof fetching, and an optional `fromDest` boolean, and the SDK handles everything from proof retrieval to Hyperbridge submission. See the [SDK overview](/developers/sdk/overview) for full `IndexerClient` documentation.

At each `AWAITING_CANCEL_TRANSACTION` yield, pass either a **serialized signed transaction** (raw bytes) or a **transaction hash** (`0x` + 64 hex chars) back via `gen.next()`. If you pass a hash, the SDK assumes the transaction has already been submitted and will wait for the receipt; if you pass raw bytes, it broadcasts first.

<Callout type="warn">
The settled status is set for both successfully filled orders **and** orders cancelled from the destination chain. `isOrderFilled` returns `true` in both cases — always check it before attempting cancellation to avoid a revert.
</Callout>

## Setup

```typescript lineNumbers title="setup.ts" icon="typescript"
import { EvmChain, IntentsCoprocessor, IntentGateway, IndexerClient, createQueryClient } from "@hyperbridge/sdk"
import { createWalletClient, http } from "viem"
import { privateKeyToAccount } from "viem/accounts"

const sourceChain = await EvmChain.create("https://eth-mainnet.g.alchemy.com/v2/demo")
const destChain   = await EvmChain.create("https://arbitrum-one.public.blastapi.io")

const coprocessor = await IntentsCoprocessor.connect("wss://nexus.ibp.network")
const intentGateway   = await IntentGateway.create(sourceChain, destChain, coprocessor)

const queryClient   = createQueryClient({ url: "https://nexus.indexer.polytope.technology" })
const indexerClient = new IndexerClient({ queryClient, source: sourceChain, dest: destChain })

const account      = privateKeyToAccount("0xYOUR_PRIVATE_KEY")
const sourceWallet = createWalletClient({ account, transport: http("https://eth-mainnet.g.alchemy.com/v2/demo") })
const destWallet   = createWalletClient({ account, transport: http("https://arbitrum-one.public.blastapi.io") })
```

## How `cancelOrder` Works

Understanding what happens on-chain helps you construct cancellations correctly.

```solidity
function cancelOrder(Order calldata order, CancelOptions calldata options) public payable;
```

The contract hashes the order to derive a unique commitment, verifies the order has not already been settled, then selects the appropriate refund path based on where the call originates.

### Data structures

```solidity
struct CancelOptions {
    uint256 relayerFee;  // Fee paid to the Hyperbridge relayer (cross-chain cancels only)
    uint256 height;      // Proof height for source-side cancels; must be > order.deadline
}
```

`height` is only meaningful for source-side cross-chain cancels. It must point to a block after the order deadline, ensuring the proof captures a state where no fill could have occurred. The SDK selects this value automatically. For same-chain and destination-side cancels, `height` is ignored.

### Access control

| Route | Before deadline | After deadline |
|---|---|---|
| Same-chain | Order creator only | Order creator only (no deadline restriction — cancel any time) |
| Source-side cross-chain | Not available — proof height must exceed deadline | Order creator only |
| Destination-side cross-chain | Order creator only | Anyone (relayers may act on behalf of expired orders) |

### Events

```solidity
// Emitted on the source chain when escrowed tokens are returned to the user.
event EscrowRefunded(bytes32 indexed commitment);
```

For same-chain cancels, `EscrowRefunded` is emitted in the same transaction. For source-side cross-chain cancels, it is emitted once Hyperbridge delivers the destination proof back to the source chain and the refund is processed. For destination-side cancels, it is emitted on the source chain once the refund message from Hyperbridge is received and settled.

## Same-Chain Orders

For same-chain orders, the contract verifies the caller is the original order creator and that tokens remain in escrow, then returns them directly. No cross-chain messaging is required and there is no deadline restriction — the order creator can cancel at any time before the order is filled.

If the order has been partially filled, the escrowed balance will already be reduced proportionally. The refund covers only what remains.

```typescript lineNumbers title="cancel-same-chain.ts" icon="typescript"
async function cancelSameChain(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled or cancelled")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient)

  // First yield: AWAITING_CANCEL_TRANSACTION
  const { value: { data, to } } = await gen.next()
  const signedTx = await sourceWallet.signTransaction({
    to,
    data,
    type: "eip1559",
  })

  // Pass signed tx back. SDK broadcasts and yields CANCELLATION_COMPLETE as the return value.
  const { value: complete } = await gen.next(signedTx)
  console.log("Refunded at:", complete!.transactionHash)
}
```

## Cross-Chain Orders

Cross-chain cancellations require Hyperbridge to either prove the order was never filled (source-side cancel) or deliver a refund message (dest-side cancel). Both paths involve a Hyperbridge dispatch fee — quote it with `quoteCancelNative` before signing. The SDK handles proof fetching, finality monitoring, and Hyperbridge submission automatically.

### Cancel from the Source Chain

Cancelling from the source chain works by proving to the source chain that the order was never filled on the destination. The contract sends a cross-chain query to the destination chain to read the fill status at a block height after the order deadline. If the result comes back empty — meaning no fill occurred — the escrowed tokens are returned. If the order was already filled, the transaction reverts.

Because the proof must come from a block after the order deadline, this path is only available once the order has expired. The SDK automatically selects the appropriate block height.

You only need to sign and submit the `cancelOrder` transaction on the source chain — the SDK fetches the destination storage proof, monitors finality on both sides, and submits everything to Hyperbridge automatically.

```typescript lineNumbers title="cancel-from-source.ts" icon="typescript"
async function cancelFromSource(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient)
  for await (const event of gen) {
    switch (event.status) {
      case "DESTINATION_FINALIZED":
        // SDK fetched a storage proof from the destination at a height > order.deadline
        console.log("Destination proof at height:", event.proof.height)
        break

      case "AWAITING_CANCEL_TRANSACTION": {
        // SDK encodes cancelOrder calldata (with the proof height embedded) and yields it here.
        // value is the quoted native fee for the GET request dispatch — include it in the tx.
        const { data, to, value } = event

        const signedTx = await sourceWallet.signTransaction({
          to,
          data,
          value,
          type: "eip1559",
        })

        // Pass the signed tx back. The generator broadcasts the tx, then yields CANCEL_STARTED.
        // gen.next() here consumes CANCEL_STARTED so the for-await resumes at SOURCE_FINALIZED.
        const { value: cancelStarted } = await gen.next(signedTx)
        if (cancelStarted?.status === "CANCEL_STARTED") {
          console.log("Cancel tx confirmed:", cancelStarted.receipt.transactionHash)
        }
        break
      }

      case "SOURCE_FINALIZED":
        // The GET request has been finalised on the source chain.
        // The SDK fetches a source proof, waits for the challenge period, and
        // submits the GET request to Hyperbridge automatically — no action needed.
        console.log("GET request finalised on source:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_DELIVERED":
        console.log("GET request delivered to Hyperbridge:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_FINALIZED":
      // metadata.calldata is the encoded handleGetResponses call to submit to the
      // ISMP handler on the source chain. This triggers onGetResponse, which
      // verifies the _filled slot and refunds the escrowed assets.
      const { calldata: handleCalldata } = event.metadata
      // Fetch the handler address from the source ISMP host
      const { handler: handlerAddress } = await sourceChain.client.readContract({
        address: sourceChain.config.host,
        abi: EvmHostABI,
        functionName: "hostParams",
      })
      const receipt = await sourceWallet.sendTransaction({
        to: handlerAddress,
        data: handleCalldata,
      })
      // Confirm the escrow was actually refunded
      const [refundLog] = parseEventLogs({
        abi: IntentGatewayV2ABI,
        eventName: "EscrowRefunded",
        logs: receipt.logs,
      })
      if (!refundLog) throw new Error("EscrowRefunded log not found — refund may have failed")
      console.log("Escrow refunded at block:", receipt.blockNumber)
      return
    }
  }
}
```

The SDK polls the destination chain until it can read the fill status at a height past the order deadline, then encodes the `cancelOrder` calldata and yields `AWAITING_CANCEL_TRANSACTION`. You sign the transaction and pass it back — the generator broadcasts it and yields `CANCEL_STARTED`. From there, the SDK handles proof fetching, waiting for the challenge period, and submitting to Hyperbridge — no further action is needed. After finalization, a relayer delivers the response to the source chain, which verifies the order was never filled and returns the escrowed tokens.

### Cancel from the Destination Chain

Cancelling from the destination chain is simpler and does not require a storage proof. The destination gateway immediately marks the order as settled — blocking any future fill attempts — and sends a refund message to the source chain via Hyperbridge. When the message arrives, the source chain authenticates it as coming from the registered destination gateway and returns the escrowed tokens to the original user.

Before the order deadline only the order creator can trigger this path. After the deadline anyone may call — this lets relayers act on behalf of users whose orders have expired without being filled, providing a recovery mechanism that doesn't require the user to be online.

<Callout type="info">
The native `value` in the `AWAITING_CANCEL_TRANSACTION` event already includes the quoted Hyperbridge dispatch fee — pass it directly to the transaction without a separate quote call.
</Callout>

```typescript lineNumbers title="cancel-from-dest.ts" icon="typescript"
async function cancelFromDest(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled or cancelled from dest")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient, true)

  // First yield: AWAITING_CANCEL_TRANSACTION — includes the quoted native fee for the dispatch
  const { value: { data, to, value } } = await gen.next()

  const signedTx = await destWallet.signTransaction({
    to,
    data,
    value,
    type: "eip1559",
  })

  // Pass signed tx back. SDK confirms the tx and yields CANCEL_STARTED.
  const { value: cancelStarted } = await gen.next(signedTx)
  if (cancelStarted?.status === "CANCEL_STARTED") {
    console.log("cancelOrder tx confirmed:", cancelStarted.receipt.transactionHash)
  }

  // Continue with the remaining status stream
  for await (const event of gen) {
    switch (event.status) {
      case "SOURCE_FINALIZED":
        // The refund message has been finalised on the destination chain.
        console.log("Refund message finalised:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_DELIVERED":
        console.log("Refund message delivered to Hyperbridge:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_FINALIZED":
        console.log("Refund message finalised on Hyperbridge:", event.metadata.transactionHash)
        break

      case "CANCELLATION_COMPLETE":
        console.log("Escrowed tokens refunded:", event.transactionHash)
        return
    }
  }
}
```

The SDK encodes the `cancelOrder` calldata for the destination gateway and yields `AWAITING_CANCEL_TRANSACTION`. You sign the transaction with sufficient `value` to cover the Hyperbridge dispatch fee and pass it back. Once confirmed, the generator yields `CANCEL_STARTED`. The gateway marks the order as settled and sends a refund message via Hyperbridge. A relayer delivers the message to the source chain, which authenticates the sender and issues the refund.

## Quoting the Cancellation Fee

Cross-chain cancellations require a Hyperbridge dispatch fee. The `cancelOrder` generator quotes the fee automatically and includes it as `value` in the `AWAITING_CANCEL_TRANSACTION` event — no separate quote call is needed. `quoteCancelNative` is available if you need the fee estimate upfront (e.g. to display it to the user before they initiate cancellation).

```typescript lineNumbers title="quote-cancel.ts" icon="typescript"
// GET request fee for source-side cancel (for display purposes)
const sourceFee = await intentGateway.quoteCancelNative(order)
console.log("Source cancel fee (wei):", sourceFee)

// POST request fee for dest-side cancel (for display purposes)
const destFee = await intentGateway.quoteCancelNative(order, true)
console.log("Dest cancel fee (wei):", destFee)
```

## Checking Order Status

```typescript lineNumbers title="check-status.ts" icon="typescript"
// Returns true if the order has been settled (filled or dest-cancelled).
const filled = await intentGateway.isOrderFilled(order)

// Returns true if the order was cancelled and escrow was returned (EscrowRefunded event exists).
const refunded = await intentGateway.isOrderRefunded(order)
```

`isOrderFilled` and `isOrderRefunded` can both be true simultaneously — a dest-cancelled order is marked as settled (so `isOrderFilled` returns `true`) and also triggers a refund (so `isOrderRefunded` returns `true` once the source chain processes the refund message). Use `isOrderRefunded` when you specifically need to confirm the user received their tokens back.

# IntentGateway

The `IntentGateway` class is the high-level facade for the IntentGatewayV2 protocol. It orchestrates the full lifecycle of an intent-based cross-chain swap: order placement, bid collection, execution, cancellation, and status checks.

```typescript lineNumbers
class IntentGateway
```

---

## Static Methods

### IntentGateway.create(source, dest, intentsCoprocessor?)

Creates a fully initialised `IntentGateway` instance. Fetches fee tokens for both chains and optionally caches solver account bytecode before returning.

```typescript lineNumbers
static async create(
  source: IEvmChain,
  dest: IEvmChain,
  intentsCoprocessor?: IntentsCoprocessor,
): Promise<IntentGateway>
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `source` | `IEvmChain` | Source chain where orders are placed and inputs are escrowed |
| `dest` | `IEvmChain` | Destination chain where solvers fill orders and deliver outputs |
| `intentsCoprocessor` | `IntentsCoprocessor` | Optional. Hyperbridge coprocessor client for bid fetching and UserOp submission |

**Returns:** `Promise<IntentGateway>`

**Example:**

```typescript lineNumbers
import { EvmChain, IntentsCoprocessor, IntentGateway } from "@hyperbridge/sdk"

const source = await EvmChain.create("https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY")
const dest = await EvmChain.create(
  "https://arb-mainnet.g.alchemy.com/v2/YOUR_KEY",
  "https://bundler.example.com",
)
const coprocessor = await IntentsCoprocessor.connect("wss://coprocessor.hyperbridge.network")

const gateway = await IntentGateway.create(source, dest, coprocessor)
```

## Methods

### execute(order, graffiti?, options)

Consumer-driven bidirectional async generator that orchestrates the full order lifecycle: placement, fee estimation, and a timed bid auction. The caller signs the placement transaction when prompted, then **selects and executes a bid itself** — on `BIDS_RECEIVED` the generator yields executable [`Bid`](#bid) objects, and the caller feeds the [`SelectBidResult`](#selectbidresult) from `bid.execute()` back via `gen.next(result)`.

For the batteries-included "pick the best and go" behaviour, use [`executeBest`](#executebestorder-graffiti-options) instead.

```typescript lineNumbers
async *execute(
  order: Order,
  graffiti: HexString = DEFAULT_GRAFFITI,
  options: {
    auctionTimeMs: number
    maxPriorityFeePerGasBumpPercent?: number
    maxFeePerGasBumpPercent?: number
    pollIntervalMs?: number
    solver?: { address: HexString; timeoutMs: number }
  },
): AsyncGenerator<IntentOrderStatusUpdate, void, HexString | SelectBidResult | undefined>
```

**Parameters:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `order` | `Order` | — | The order to place and execute. `order.fees` may be `0`; it is estimated automatically if so. |
| `graffiti` | `HexString` | `0x000...0` | Optional bytes32 tag for orderflow attribution/revenue share |
| `options.auctionTimeMs` | `number` | — | **Required.** Duration in ms to collect bids before surfacing them for selection |
| `options.maxPriorityFeePerGasBumpPercent` | `number` | `8` | Bump percentage applied to priority fee estimate |
| `options.maxFeePerGasBumpPercent` | `number` | `10` | Bump percentage applied to max fee estimate |
| `options.pollIntervalMs` | `number` | — | Interval between bid-polling attempts |
| `options.solver` | `{ address, timeoutMs }` | — | Restrict bids to `address` until `timeoutMs` elapses, then accept any solver |

**Yields:** `IntentOrderStatusUpdate`

**Receives (via `gen.next(value)`):** the signed placement tx (hash or raw tx) after `AWAITING_PLACE_ORDER`, and a `SelectBidResult` after `BIDS_RECEIVED` (or `undefined` to keep polling for fresh bids).

**Example:**

```typescript lineNumbers
const signer = createWalletClient({ ... })

// execute is a bidirectional generator — hold a reference to pass values back
const gen = gateway.execute(order, DEFAULT_GRAFFITI, { auctionTimeMs: 15_000 })

let input: HexString | SelectBidResult | undefined
while (true) {
  const { value: update, done } = await gen.next(input)
  input = undefined
  if (done) break

  switch (update.status) {
    case "AWAITING_PLACE_ORDER": {
      // Sign and broadcast the placement transaction, then feed the hash back.
      input = await signer.sendTransaction({
        to: update.to,
        data: update.data,
        value: update.value,
      })
      break
    }
    case "BIDS_RECEIVED": {
      // You own selection. Rank the bids however you like, simulate, execute.
      const sorted = await gateway.sortBids(order, update.bids)
      for (const bid of sorted) {
        try {
          await bid.simulate()
          input = await bid.execute() // feed the SelectBidResult back to the generator
          break
        } catch {
          // bid failed simulation/execution — try the next one
        }
      }
      // leaving `input` undefined keeps the auction polling for fresh bids
      break
    }
    case "BID_SELECTED":
      console.log("Solver selected:", update.selectedSolver)
      break
    case "FILLED":
      console.log("Order filled!")
      break
  }
}
```

### quoteIntent(params)

Quotes an intent using the SDK's intent quote strategies. The only supported strategy today is `uniswap_v4`, which always prices against the configured Base (`EVM-8453`) pool rather than the gateway destination chain. The method is strategy-shaped so future quote sources can be added without changing partner integrations.

Use `amountIn` and `amountOut` when constructing the `inputs` and `output.assets` for an IntentGateway V2 order.

```typescript lineNumbers
const gateway = await IntentGateway.create(source, dest)

const quote = await gateway.quoteIntent({
  tokenIn: {
    address: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
    decimals: 6,
    symbol: "USDC",
  },
  tokenOut: {
    address: "0x46C85152bFe9f96829aA94755D9f915F9B10EF5F",
    decimals: 6,
    symbol: "cNGN",
  },
  amountIn: 100_000_000n,
})

console.log(quote.amountIn, quote.amountOut)
```

For pools not configured by the SDK, pass an explicit Uniswap V4 PoolKey:

```typescript lineNumbers
const quote = await gateway.quoteIntent({
  tokenIn,
  tokenOut,
  amountOut: 50_000_000n,
  uniswapV4: {
    poolKey: {
      currency0: "0x...",
      currency1: "0x...",
      fee: 1500,
      tickSpacing: 30,
      hooks: "0x0000000000000000000000000000000000000000",
      quoterAddress: "0x0d5e0f971ed27fbff6c2837bf31316121532048d",
    },
  },
})
```

When passing an explicit PoolKey for cross-chain quotes, also pass `currencyIn` inside the `poolKey` override — the Base-side address of the input currency. It defaults to `tokenIn.address`, which only works when that address is part of the Base pool, and must equal `currency0` or `currency1`.

The result includes `amountIn`, `amountOut`, and quote metadata with the quote chain, V4 PoolKey, quoter address, and the IntentGateway protocol fee on the source chain (`protocolFeeBps`).

`amountIn` and `amountOut` already account for the IntentGateway protocol fee that the gateway deducts from order inputs. Exact-input quotes price the swap against the post-fee input, so `amountOut` is what you can expect after the fee; exact-output quotes return the gross `amountIn` you must supply so that, after the fee, enough remains to produce the requested `amountOut`. Use these amounts directly as the order's `inputs` and `output.assets` — no further fee adjustment is required. Apply only your own slippage tolerance:

```typescript lineNumbers
// `amountOut` already nets the gateway fee; add your slippage tolerance.
const minAmountOut = (quote.amountOut * (10_000n - SLIPPAGE_BPS)) / 10_000n
```

### executeBest(order, graffiti?, options)

Batteries-included variant of [`execute`](#executeorder-graffiti-options): places the order, then auto-selects and executes the best bid each round (sort → simulate → sign → submit) with no bid-selection input from the caller. The caller still signs the placement transaction (`AWAITING_PLACE_ORDER`); every other stage is handled automatically and surfaced for observation. If a round's bids all fail, it keeps polling for fresh bids until the deadline.

```typescript lineNumbers
async *executeBest(
  order: Order,
  graffiti: HexString = DEFAULT_GRAFFITI,
  options: {
    auctionTimeMs: number
    maxPriorityFeePerGasBumpPercent?: number
    maxFeePerGasBumpPercent?: number
    pollIntervalMs?: number
    solver?: { address: HexString; timeoutMs: number }
  },
): AsyncGenerator<IntentOrderStatusUpdate, void, HexString>
```

**Example:**

```typescript lineNumbers
const gen = gateway.executeBest(order, DEFAULT_GRAFFITI, { auctionTimeMs: 15_000 })
for await (const update of gen) {
  if (update.status === "AWAITING_PLACE_ORDER") {
    const txHash = await signer.sendTransaction({ to: update.to, data: update.data, value: update.value })
    await gen.next(txHash) // only the placement step needs feedback
  } else if (update.status === "FILLED") {
    console.log("Order filled!")
  }
}
```

### resume(order, options) / resumeBest(order, options)

Resume execution of a previously placed order (after an app restart or crash). The order must already be on-chain — it must have a valid `id` and `session`. There is no placement step, so a plain `for await` loop is sufficient.

- `resume` is consumer-driven, mirroring [`execute`](#executeorder-graffiti-options) (you select bids and feed back `SelectBidResult`).
- `resumeBest` is the autopilot, mirroring [`executeBest`](#executebestorder-graffiti-options).

```typescript lineNumbers
async *resume(order: Order, options: ResumeIntentOrderOptions):
  AsyncGenerator<IntentOrderStatusUpdate, void, SelectBidResult | undefined>

async *resumeBest(order: Order, options: ResumeIntentOrderOptions):
  AsyncGenerator<IntentOrderStatusUpdate, void>
```

`ResumeIntentOrderOptions` is `{ sessionPrivateKey?, auctionTimeMs, pollIntervalMs?, solver? }`.

### quoteUniswap(params)

> Standalone export (`import { quoteUniswap } from "@hyperbridge/sdk"`), not a method on `IntentGateway`.

Quotes a same-chain Uniswap swap that can be used when constructing the input or output side of an intent order. The helper checks Uniswap V2, V3, and V4 by default; pass `protocols: ["v4"]` to quote Uniswap V4 only.

```typescript lineNumbers
import { quoteUniswap } from "@hyperbridge/sdk"

const result = await quoteUniswap({
  chainId: 8453,
  tokenIn: {
    address: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
    decimals: 6,
    symbol: "USDC",
    chainId: 8453,
  },
  tokenOut: {
    address: "0x0000000000000000000000000000000000000000",
    decimals: 18,
    symbol: "ETH",
    chainId: 8453,
  },
  amountIn: 100_000_000n,
  tradeType: "EXACT_INPUT",
  protocols: ["v4"],
})

console.log(result.bestQuote?.amountOut)
```

Set `recipient` to include executable transaction calldata in each quote. For exact-output swaps, use `tradeType: "EXACT_OUTPUT"` with `amountOut` instead of `amountIn`.

### cancelOrder(order, indexerClient, options?)

Async generator that cancels a placed order and streams status events until cancellation is complete. Supports both source-chain and destination-chain initiated cancellation.

```typescript lineNumbers
async *cancelOrder(
  order: Order,
  indexerClient: IsmpClient,
  options?: { from?: "source" | "destination" },
): AsyncGenerator<CancelEvent>
```

**Parameters:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `order` | `Order` | — | The order to cancel |
| `indexerClient` | `IsmpClient` | — | Indexer client for streaming ISMP request status |
| `options` | `{ from?: "source" \| "destination" }` | `{}` | Pass `{ from: "destination" }` to initiate on the destination chain. |

**Yields:** `CancelEvent`

**Example:**

```typescript lineNumbers
const gen = gateway.cancelOrder(order, indexerClient)
for await (const event of gen) {
  switch (event.status) {
    case "DESTINATION_FINALIZED":
      console.log("Destination proof ready at height:", event.proof.height)
      break
    case "AWAITING_CANCEL_TRANSACTION": {
      // cancelOrder is a bidirectional generator — pass the signed tx or tx hash back.
      const txHash = await signer.sendTransaction({
        to: event.to,
        data: event.data,
        value: event.value,
      })
      // gen.next(txHash) resumes the generator; CANCEL_STARTED is consumed here.
      const { value: cancelStarted } = await gen.next(txHash)
      if (cancelStarted?.status === "CANCEL_STARTED") {
        console.log("Cancel tx confirmed:", cancelStarted.receipt.transactionHash)
      }
      break
    }
    case "SOURCE_FINALIZED":
      console.log("Cancel request finalised on source:", event.metadata.transactionHash)
      break
    case "HYPERBRIDGE_DELIVERED":
      console.log("Cancel message delivered to Hyperbridge:", event.metadata.transactionHash)
      break
    case "HYPERBRIDGE_FINALIZED":
      console.log("Cancel message finalised on Hyperbridge:", event.metadata.transactionHash)
      break
    case "CANCELLATION_COMPLETE":
      console.log("Refunded in tx:", event.transactionHash)
      break
  }
}
```

## Types

### IntentOrderStatusUpdate

Tagged union of all status updates yielded by the `execute` generator. Use the `status` field to discriminate.

```typescript lineNumbers
type IntentOrderStatusUpdate =
  | { status: "AWAITING_PLACE_ORDER"; to: HexString; data: HexString; value?: bigint; sessionPrivateKey: HexString }
  | { status: "ORDER_PLACED"; order: Order; receipt: TransactionReceipt }
  | { status: "AWAITING_BIDS"; commitment: HexString; totalFilledAssets: TokenInfo[]; remainingAssets: TokenInfo[] }
  | { status: "NEW_BID"; commitment: HexString; bid: Bid }
  | { status: "BIDS_RECEIVED"; commitment: HexString; bidCount: number; bids: Bid[] }
  | { status: "BID_SELECTED"; commitment: HexString; selectedSolver: HexString; userOpHash: HexString; userOp: PackedUserOperation; transactionHash?: HexString }
  | { status: "FILLED"; commitment: HexString; userOpHash: HexString; selectedSolver: HexString; transactionHash?: HexString; totalFilledAssets: TokenInfo[]; remainingAssets: TokenInfo[] }
  | { status: "PARTIAL_FILL"; commitment: HexString; userOpHash: HexString; selectedSolver: HexString; transactionHash?: HexString; filledAssets: TokenInfo[]; totalFilledAssets: TokenInfo[]; remainingAssets: TokenInfo[] }
  | { status: "EXPIRED"; commitment: HexString; totalFilledAssets?: TokenInfo[]; remainingAssets?: TokenInfo[]; error: string }
  | { status: "FAILED"; commitment?: HexString; totalFilledAssets?: TokenInfo[]; remainingAssets?: TokenInfo[]; error: string }
```

| Status | Fields | Description |
|--------|--------|-------------|
| `AWAITING_PLACE_ORDER` | `to`, `data`, `value?`, `sessionPrivateKey` | Caller must sign and broadcast the placement transaction, then send back the tx hash |
| `ORDER_PLACED` | `order`, `receipt` | Order confirmed on-chain; `receipt` is the full viem `TransactionReceipt` of the placement transaction |
| `AWAITING_BIDS` | `commitment`, `totalFilledAssets`, `remainingAssets` | Polling the coprocessor for solver bids |
| `NEW_BID` | `commitment`, `bid` | A previously unseen bid arrived during the auction (informational; one [`Bid`](#bid)) |
| `BIDS_RECEIVED` | `commitment`, `bidCount`, `bids` | Auction round complete; `bids` is an array of executable [`Bid`](#bid) objects to select from |
| `BID_SELECTED` | `commitment`, `selectedSolver`, `userOpHash`, `userOp`, `transactionHash?` | Best bid selected and UserOperation submitted to the bundler |
| `FILLED` | `commitment`, `userOpHash`, `selectedSolver`, `transactionHash?`, `totalFilledAssets`, `remainingAssets` | Order fully filled on the destination chain |
| `PARTIAL_FILL` | `commitment`, `userOpHash`, `selectedSolver`, `transactionHash?`, `filledAssets`, `totalFilledAssets`, `remainingAssets` | Order partially filled; more fills may follow |
| `EXPIRED` | `commitment`, `totalFilledAssets?`, `remainingAssets?`, `error` | Order deadline reached or no new bids available — terminal |
| `FAILED` | `commitment?`, `totalFilledAssets?`, `remainingAssets?`, `error` | Retryable error (e.g. bundler rejection, simulation failure) — executor retries automatically |

### Bid

A first-class, executable solver bid surfaced via the `NEW_BID` and `BIDS_RECEIVED` status updates. Each `Bid` wraps a single decoded `FillerBid` and exposes the metadata needed to rank it alongside the two primitives required to act on it: `simulate()` to pre-check it off-chain, and `execute()` to sign the solver selection and submit the UserOperation to the bundler.

```typescript lineNumbers
interface Bid {
  /** The solver account that submitted this bid (`userOp.sender`). */
  readonly solverAddress: HexString
  /** Decoded fill outputs — the tokens and amounts the solver offers. */
  readonly outputs: TokenInfo[]
  /** Relayer fee from the decoded fill options. */
  readonly relayerFee: bigint
  /** Hyperbridge native dispatch fee from the decoded fill options. */
  readonly nativeDispatchFee: bigint
  /** Raw UserOperation for advanced consumers (custom batching, paymaster flows). */
  readonly userOp: PackedUserOperation

  /** Runs the batched `select` + `fillOrder` `eth_call` simulation. Throws if it reverts. */
  simulate(): Promise<void>
  /** Signs the `SelectSolver` message, packs the signature, and submits the UserOp to the bundler. */
  execute(): Promise<SelectBidResult>
  /** Prices the bid's outputs in USD. Returns `null` when any output token cannot be priced. */
  outputUsdValue(): Promise<Decimal | null>
}
```

The session-key signature over the `SelectSolver` message is resolved lazily and cached, so it is signed at most once whether you call `simulate()`, `execute()`, or both.


### CancelEvent

Tagged union of all status updates yielded by the `cancelOrder` generator.

```typescript lineNumbers
type BlockMetadata = {
  blockHash: string
  blockNumber: number
  transactionHash: string
  timestamp?: number
}

type CancelEvent =
  | { status: "DESTINATION_FINALIZED"; proof: IProof }
  | { status: "AWAITING_CANCEL_TRANSACTION"; data: HexString; to: HexString; value: bigint }
  | { status: "CANCEL_STARTED"; receipt: TransactionReceipt }
  | { status: "SOURCE_FINALIZED"; metadata: BlockMetadata }
  | { status: "HYPERBRIDGE_DELIVERED"; metadata: BlockMetadata }
  | { status: "HYPERBRIDGE_FINALIZED"; metadata: BlockMetadata }
  | { status: "CANCELLATION_COMPLETE"; blockNumber: number; transactionHash: HexString }
```

| Status | Fields | Description |
|--------|--------|-------------|
| `DESTINATION_FINALIZED` | `proof` | State proof from the destination chain is ready |
| `AWAITING_CANCEL_TRANSACTION` | `data`, `to`, `value` | Caller must sign and submit the cancel transaction |
| `CANCEL_STARTED` | `receipt` | Cancel transaction confirmed on-chain |
| `SOURCE_FINALIZED` | `metadata` | Cancel request finalised on source chain |
| `HYPERBRIDGE_DELIVERED` | `metadata` | Cancel message delivered to Hyperbridge |
| `HYPERBRIDGE_FINALIZED` | `metadata` | Cancel message finalised on Hyperbridge |
| `CANCELLATION_COMPLETE` | `blockNumber`, `transactionHash` | Escrow refunded; cancellation done |

### Order

Represents a cross-chain swap intent submitted to IntentGatewayV2.

```typescript lineNumbers
interface Order {
  id?: string          // Order commitment hash (populated after placement)
  user: HexString      // Address of the order creator
  source: HexString    // State-machine ID of the source chain (e.g. "EVM-1")
  destination: HexString // State-machine ID of the destination chain (e.g. "EVM-42161")
  deadline: bigint     // Unix timestamp after which the order can be cancelled
  nonce: bigint        // Unique order nonce (set by contract on placement)
  fees: bigint         // Protocol fee in fee-token units
  session: HexString   // Ephemeral session key address (set during placement)
  predispatch: DispatchInfo // Optional calls to execute on the source chain before escrowing
  inputs: TokenInfo[]  // Tokens escrowed on the source chain
  output: PaymentInfo  // Token the solver must deliver on the destination chain
}
```

# EvmChain

The `EvmChain` class provides methods for interacting with EVM-compatible chains (Ethereum, Arbitrum, Optimism, Base, etc.) in the Hyperbridge protocol. It handles proof generation, state queries, and cross-chain message verification.

```typescript lineNumbers
class EvmChain implements IChain
```

## Static Methods

### EvmChain.create(rpcUrl, bundlerUrl?)

Auto-detects the chain ID from the RPC endpoint and resolves the correct `IsmpHost` contract address for that chain. This is the recommended way to instantiate `EvmChain` for known Hyperbridge deployments.

```typescript lineNumbers
static async create(rpcUrl: string, bundlerUrl?: string): Promise<EvmChain>
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `rpcUrl` | `string` | HTTP(S) RPC URL of the EVM node |
| `bundlerUrl` | `string` | Optional. ERC-4337 bundler URL for account abstraction support |

**Returns:** `Promise<EvmChain>` — A fully initialized `EvmChain` instance

**Throws:** If the chain ID returned by the RPC is not a known Hyperbridge deployment

**Example:**

```typescript lineNumbers
import { EvmChain } from "@hyperbridge/sdk"

// Auto-detect chain from RPC
const chain = await EvmChain.create("https://mainnet.base.org")

// With ERC-4337 bundler for IntentGateway
const destChain = await EvmChain.create(
  "https://arb-mainnet.g.alchemy.com/v2/YOUR_KEY",
  "https://bundler.example.com"
)
```

### EvmChain.fromParams(params)

Creates an `EvmChain` instance from a fully-specified params object. Use this when you already know the chain ID and `IsmpHost` address and don't need auto-detection.

```typescript lineNumbers
static fromParams(params: EvmChainParams): EvmChain
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `params.chainId` | `number` | The EVM chain ID (e.g., 1 for Ethereum, 42161 for Arbitrum) |
| `params.rpcUrl` | `string` | RPC URL for the chain (must be an archive node for proof queries) |
| `params.host` | `HexString` | Address of the IsmpHost contract on this chain |
| `params.consensusStateId` | `string` | Optional. Consensus state identifier on Hyperbridge (auto-detected for known chains) |
| `params.bundlerUrl` | `string` | Optional. ERC-4337 bundler URL for account abstraction support |

**Returns:** `EvmChain`

**Example:**

```typescript lineNumbers
import { EvmChain } from "@hyperbridge/sdk"

const ethereumChain = EvmChain.fromParams({
  chainId: 1,
  rpcUrl: "https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY",
  host: "0x87ea459...",
  consensusStateId: "ETH0" // Optional, auto-detected for known chains
})

const arbitrumChain = EvmChain.fromParams({
  chainId: 42161,
  rpcUrl: "https://arb-mainnet.g.alchemy.com/v2/YOUR_KEY",
  host: "0x42fa123...",
  bundlerUrl: "https://bundler.example.com" // Optional, for ERC-4337 support
})
```

### queryRequestReceipt()

Queries the request receipt to determine if a request has been delivered and by whom.

```typescript lineNumbers
async queryRequestReceipt(commitment: HexString): Promise<HexString | undefined>
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `commitment` | `HexString` | The request commitment hash |

**Returns:** `Promise<HexString | undefined>` - The relayer address that delivered the request, or `undefined` if not yet delivered

**Example:**

```typescript lineNumbers
const relayer = await evmChain.queryRequestReceipt(commitmentHash)
if (relayer) {
  console.log(`Request delivered by: ${relayer}`)
} else {
  console.log("Request not yet delivered")
}
```

### queryProof()

Queries cryptographic proofs for cross-chain message verification.

```typescript lineNumbers
async queryProof(
  message: IMessage, 
  counterparty: string, 
  at?: bigint
): Promise<HexString>
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `message` | `IMessage` | The message containing requests or responses |
| `counterparty` | `string` | The counterparty chain's state machine ID |
| `at` | `bigint` | Optional. Block number to query at (defaults to latest) |

**Returns:** `Promise<HexString>` - Encoded EVM state proof

**Note:** This generates Merkle-Patricia trie proofs for the EVM state

---

### queryStateProof()

Queries storage proofs for specific keys at a given block height.

```typescript lineNumbers
async queryStateProof(
  at: bigint, 
  keys: HexString[], 
  address?: HexString

```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `at` | `bigint` | Block number to query at |
| `keys` | `HexString[]` | Storage keys to fetch proofs for |
| `address` | `HexString` | Optional. Contract address (defaults to host contract) |

**Returns:** `Promise<HexString>` - Encoded storage proof

**Example:**

```typescript lineNumbers

  18000000n,
  ["0x1234...", "0x5678..."],
  "0xContractAddress..."
)
```

### getTransactionReceipt()

Waits for an already-submitted transaction to be confirmed and returns its receipt. Use this when you have already broadcast the transaction yourself and only have the hash.

```typescript lineNumbers
async getTransactionReceipt(hash: HexString): Promise<TransactionReceipt>
```

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `hash` | `HexString` | The 32-byte transaction hash (`0x` + 64 hex characters) |

**Returns:** `Promise<TransactionReceipt>` - The transaction receipt once confirmed

**Example:**

```typescript lineNumbers
// You already submitted the transaction externally
const txHash = "0xabc123..."
const receipt = await evmChain.getTransactionReceipt(txHash)
console.log(`Confirmed in block ${receipt.blockNumber}`)
```

## Important Notes

### Archive Node Requirement

**Critical:** The RPC URL must point to an archive node. Regular nodes only retain recent state and cannot provide storage proofs for older blocks, which are essential for:
- Timeout proof generation
- Historical state verification
- Cross-chain message validation

### Consensus State IDs

Consensus state IDs identify how a chain is verified on Hyperbridge:
- Chains using Ethereum consensus share `ETH0`
- Polygon uses `POLY`
- BSC uses `BSC0`
- Gnosis uses `GNO0`

For custom chains, specify the appropriate `consensusStateId` based on how the chain is configured on Hyperbridge.

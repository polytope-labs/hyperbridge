# HyperbridgeLzEndpoint

`HyperbridgeLzEndpoint` is a drop-in replacement for the LayerZero V2 endpoint that routes all cross-chain messages through Hyperbridge's ISMP protocol. Existing OFTs and OApps can point to this contract as their endpoint to use Hyperbridge for cross-chain transport without any code changes.

The contract implements `ILayerZeroEndpointV2` for full OApp/OFT compatibility and inherits [HyperApp](/developers/evm/api/hyperapp) for ISMP message handling.

## How It Works

When an OApp calls `send()` on the endpoint, the adapter encodes the LayerZero message into an ISMP POST request body and dispatches it through Hyperbridge. On the destination chain, the same adapter receives the ISMP message via `onAccept()`, decodes it back into LayerZero format, and delivers it to the destination OApp via `lzReceive()`.

The adapter assumes the same contract address on all chains via CREATE2 deployment. It verifies incoming messages by checking that `request.from` matches its own address — if it's the same contract on both chains, only legitimate messages from the adapter on the source chain are accepted.

### Compose Support

The adapter supports LayerZero's two-step compose pattern. When an OApp calls `sendCompose()` during `lzReceive()`, the message hash is queued. A relayer or executor then calls `lzCompose()` with the full message, which is verified against the queued hash before being delivered to the target `ILayerZeroComposer`.

### Fee Model

Fees are paid in the host's fee token or native token, consistent with Hyperbridge's standard fee model. The adapter charges a per-destination relayer fee (configurable by the owner) on top of the protocol fee. The default relayer fee is $0.30, derived from the fee token's decimals.

## Deployment

<Steps>
<Step>
### Deploy with CREATE2

The adapter must have the same address on all chains so it can verify cross-chain message sources.

```solidity lineNumbers
import {HyperbridgeLzEndpoint} from "@hyperbridge/lz-endpoint/HyperbridgeLzEndpoint.sol";

bytes32 salt = keccak256("hyperbridge-lz-v1");
HyperbridgeLzEndpoint endpoint = new HyperbridgeLzEndpoint{salt: salt}(admin);
```
</Step>
<Step>
### Configure the host and local EID

```solidity lineNumbers
endpoint.setHost(ISMP_HOST_ADDRESS, LOCAL_LZ_EID);
```

This also sets the default relayer fee to $0.30 based on the host's fee token decimals.
</Step>
<Step>
### Register EID mappings

Map each LayerZero endpoint ID to its corresponding ISMP state machine identifier:

```solidity lineNumbers
import {StateMachine} from "@hyperbridge/core/libraries/StateMachine.sol";

// Ethereum = LZ EID 30001, ISMP EVM-1
endpoint.setEidMapping(30001, StateMachine.evm(1));

// Arbitrum = LZ EID 30110, ISMP EVM-42161
endpoint.setEidMapping(30110, StateMachine.evm(42161));

// Base = LZ EID 30184, ISMP EVM-8453
endpoint.setEidMapping(30184, StateMachine.evm(8453));
```
</Step>
</Steps>

## Migrating an OFT

To migrate an existing OFT from LayerZero to Hyperbridge:

<Steps>
<Step>
### Point the OFT to the adapter

Update the OFT's endpoint address to the `HyperbridgeLzEndpoint` deployment. For OFTs using the standard `OApp` base, this is set in the constructor or via `setEndpoint()`.
</Step>
<Step>
### Set peers

Configure the OFT's peers as usual via `setPeer(dstEid, peerAddress)`. The adapter handles routing — the OFT doesn't need to know about ISMP.
</Step>
<Step>
### Send as normal

The OFT's `send()` call goes through the adapter transparently:

```solidity lineNumbers
// This works exactly the same as with the standard LZ endpoint
oft.send(
    SendParam({
        dstEid: 30110,                          // Arbitrum
        to: bytes32(uint256(uint160(recipient))),
        amountLD: 1 ether,
        minAmountLD: 1 ether,
        extraOptions: "",
        composeMsg: "",
        oftCmd: ""
    }),
    MessagingFee({nativeFee: quotedFee, lzTokenFee: 0}),
    msg.sender
);
```
</Step>
</Steps>

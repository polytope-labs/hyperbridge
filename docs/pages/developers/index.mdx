# Hyperbridge

For the first time, developers can build mission-critical cross-chain applications powered by consensus proofs. Hyperbridge is a fully chain-agnostic protocol that enables developers to build applications that can securely interoperate with any blockchain.

## Getting Started

If you're new to Hyperbridge and its workings, we suggest starting with the [Hyperbridge Protocol Specification](notion://www.notion.so/protocol). After gaining a basic understanding, you can delve into the explore section to discover the protocol's diverse components. Once familiar, you can begin creating your own applications using the Hyperbridge SDK, which we offer for various languages and platforms.

## Solidity SDK

We provide a Solidity SDK that allows your solidity dAPPs to send and receive cross-chain messages through Hyperbridge. This SDK is designed to be easy to use and provides a simple interface for interacting with the Hyperbridge protocol.

## Polkadot SDK

We provide the ISMP SDK that allows substrate-based chains to send and receive cross-chain messages through Hyperbridge. This SDK is designed to be easy to use and provides a simple interface for interacting with the Hyperbridge protocol.

## Network Operators

Network operators are responsible for running the Hyperbridge network. They are responsible for relaying messages, and producing blocks. Network operators are incentivized to run the network by receiving fees from users who wish to perform cross-chain operations.

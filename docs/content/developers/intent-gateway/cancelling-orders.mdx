# Cancelling Orders

When a user wants to reclaim their escrowed tokens — because no solver filled the order, or the order expired — they call `cancelOrder` on the `IntentGatewayV2` contract. The contract computes the commitment, checks that it has not already been filled or cancelled (`_filled[commitment] == address(0)`), then routes to one of three internal handlers depending on where the call is made relative to the order's source and destination chains.

The `IntentGateway.cancelOrder()` SDK method abstracts over all three paths. You pass the order, an `IndexerClient` for proof fetching, and an optional `fromDest` boolean, and the SDK handles everything from proof retrieval to Hyperbridge submission. See the [SDK overview](/developers/sdk/overview) for full `IndexerClient` documentation.

At each `AWAITING_CANCEL_TRANSACTION` yield, pass either a **serialized signed transaction** (raw bytes) or a **transaction hash** (`0x` + 64 hex chars) back via `gen.next()`. If you pass a hash, the SDK assumes the transaction has already been submitted and will wait for the receipt; if you pass raw bytes, it broadcasts first.

<Callout type="warn">
`_filled[commitment]` is set to a non-zero address for both successfully filled orders **and** orders cancelled from the destination chain. `isOrderFilled` returns `true` in both cases — always check it before attempting cancellation to avoid a `Filled()` revert.
</Callout>

## Setup

```typescript lineNumbers title="setup.ts" icon="typescript"
import { EvmChain, IntentsCoprocessor, IntentGateway, IndexerClient, createQueryClient } from "@hyperbridge/sdk"
import { createWalletClient, http } from "viem"
import { privateKeyToAccount } from "viem/accounts"

const sourceChain = await EvmChain.create("https://eth-mainnet.g.alchemy.com/v2/demo")
const destChain   = await EvmChain.create("https://arbitrum-one.public.blastapi.io")

const coprocessor = await IntentsCoprocessor.connect("wss://nexus.ibp.network")
const intentGateway   = await IntentGateway.create(sourceChain, destChain, coprocessor)

const queryClient   = createQueryClient({ url: "https://nexus.indexer.polytope.technology" })
const indexerClient = new IndexerClient({ queryClient, source: sourceChain, dest: destChain })

const account      = privateKeyToAccount("0xYOUR_PRIVATE_KEY")
const sourceWallet = createWalletClient({ account, transport: http("https://eth-mainnet.g.alchemy.com/v2/demo") })
const destWallet   = createWalletClient({ account, transport: http("https://arbitrum-one.public.blastapi.io") })
```

## Same-Chain Orders

For same-chain orders (`source == destination`), `_cancelSameChain` verifies that the caller is the original order creator and that the commitment's escrow balance is non-zero, then refunds all remaining escrowed tokens directly via `_withdraw`. No cross-chain messaging is required and there is no deadline restriction — the order creator can cancel at any time before the order is filled.

If the order has been partially filled, the escrow balance will already be reduced proportionally. The refund covers only the remaining balance, not the full original amount.

```typescript lineNumbers title="cancel-same-chain.ts" icon="typescript"
async function cancelSameChain(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled or cancelled")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient)

  // First yield: AWAITING_CANCEL_TRANSACTION
  const { value: { data, to } } = await gen.next()
  const signedTx = await sourceWallet.signTransaction({
    to,
    data,
    type: "eip1559",
  })

  // Pass signed tx back. SDK broadcasts and yields CANCELLATION_COMPLETE as the return value.
  const { value: complete } = await gen.next(signedTx)
  console.log("Refunded at:", complete!.transactionHash)
}
```

## Cross-Chain Orders

Cross-chain cancellations require Hyperbridge to either prove the order was never filled (source-side cancel) or deliver a refund message (dest-side cancel). Both paths involve a Hyperbridge dispatch fee — quote it with `quoteCancelNative` before signing. The SDK handles proof fetching, finality monitoring, and Hyperbridge submission automatically.

### Cancel from the Source Chain

Cancelling from the source chain works by cryptographically proving to the source chain that the order was never filled on the destination. The contract dispatches a Hyperbridge `GET` request targeting the `_filled[commitment]` storage slot on the destination gateway. When the response arrives, `onGetResponse` inspects the slot: an empty value triggers `_withdraw` and emits `EscrowRefunded`; a non-empty value means the order was already filled and reverts with `Filled()`.

Because the proof height must be taken from a block **after** the order deadline (`options.height > order.deadline`), this path is only available once the order has expired — it guarantees that no fill could legitimately have occurred at or after that height. The SDK automatically selects the proof height.

The `GET` request encodes the `_filled` storage slot as the key and the `WithdrawalRequest` as context, so `onGetResponse` knows which commitment and tokens to refund when the response arrives. You only need to sign and submit the `cancelOrder` transaction on the source chain — the SDK fetches the destination storage proof, monitors finality on both sides, and submits the proof to Hyperbridge automatically.

```typescript lineNumbers title="cancel-from-source.ts" icon="typescript"
async function cancelFromSource(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient)
  for await (const event of gen) {
    switch (event.status) {
      case "DESTINATION_FINALIZED":
        // SDK fetched a storage proof from the destination at a height > order.deadline
        console.log("Destination proof at height:", event.proof.height)
        break

      case "AWAITING_CANCEL_TRANSACTION": {
        // SDK encodes cancelOrder calldata (with the proof height embedded) and yields it here.
        // value is the quoted native fee for the GET request dispatch — include it in the tx.
        const { data, to, value } = event

        const signedTx = await sourceWallet.signTransaction({
          to,
          data,
          value,
          type: "eip1559",
        })

        // Pass the signed tx back. The generator broadcasts the tx, then yields CANCEL_STARTED.
        // gen.next() here consumes CANCEL_STARTED so the for-await resumes at SOURCE_FINALIZED.
        const { value: cancelStarted } = await gen.next(signedTx)
        if (cancelStarted?.status === "CANCEL_STARTED") {
          console.log("Cancel tx confirmed:", cancelStarted.receipt.transactionHash)
        }
        break
      }

      case "SOURCE_FINALIZED":
        // The GET request has been finalised on the source chain.
        // The SDK fetches a source proof, waits for the challenge period, and
        // submits the GET request to Hyperbridge automatically — no action needed.
        console.log("GET request finalised on source:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_DELIVERED":
        console.log("GET request delivered to Hyperbridge:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_FINALIZED":
        // GET request finalised on Hyperbridge. A relayer will deliver the response
        // to the source chain, which triggers onGetResponse to check the _filled slot
        // and refund the escrowed tokens. No action required from the caller.
        console.log("GET request finalised on Hyperbridge:", event.metadata.transactionHash)
        break
    }
  }
}
```

The SDK polls the destination chain indexer until a committed state height exceeds `order.deadline`, then fetches the `_filled[commitment]` storage proof at that height and yields `DESTINATION_FINALIZED`. It then encodes the `cancelOrder` calldata (with the proof height embedded) and yields `AWAITING_CANCEL_TRANSACTION`. You sign the transaction and pass it back via `gen.next(signedTx)` — the generator broadcasts the transaction and yields `CANCEL_STARTED`. The `for await` loop then resumes at `SOURCE_FINALIZED`, at which point the SDK internally fetches a source-chain proof, waits for the challenge period, and submits the GET request to Hyperbridge — no action required. The loop continues with `HYPERBRIDGE_DELIVERED` and `HYPERBRIDGE_FINALIZED`. After finalization, a relayer delivers the `handleGetResponses` call to the source chain, which triggers `onGetResponse` to verify the `_filled` slot and refund the escrow.

### Cancel from the Destination Chain

Cancelling from the destination chain is simpler and does not require a storage proof. The destination gateway immediately sets `_filled[commitment] = address(order.user)` — blocking any future fill attempts — and dispatches a `RefundEscrow` POST message to the source chain via Hyperbridge. When the message arrives, `onAccept` on the source chain authenticates it against the registered destination gateway address, decodes the `WithdrawalRequest`, and calls `_withdraw(isRefund=true)` to return the escrowed tokens to `order.user`.

Before the order deadline only the order creator can trigger this path. After the deadline anyone may call — this lets relayers act on behalf of users whose orders have expired without being filled, providing a recovery mechanism that doesn't require the user to be online.

<Callout type="info">
The native `value` in the `AWAITING_CANCEL_TRANSACTION` event already includes the quoted Hyperbridge dispatch fee for the `RefundEscrow` POST message — pass it directly to the transaction without a separate quote call.
</Callout>

```typescript lineNumbers title="cancel-from-dest.ts" icon="typescript"
async function cancelFromDest(order: Order) {
  if (await intentGateway.isOrderFilled(order)) {
    throw new Error("Order already filled or cancelled from dest")
  }

  const gen = intentGateway.cancelOrder(order, indexerClient, true)

  // First yield: AWAITING_CANCEL_TRANSACTION — includes the quoted native fee for the dispatch
  const { value: { data, to, value } } = await gen.next()

  const signedTx = await destWallet.signTransaction({
    to,
    data,
    value,
    type: "eip1559",
  })

  // Pass signed tx back. SDK confirms the tx and yields CANCEL_STARTED.
  const { value: cancelStarted } = await gen.next(signedTx)
  if (cancelStarted?.status === "CANCEL_STARTED") {
    console.log("cancelOrder tx confirmed:", cancelStarted.receipt.transactionHash)
  }

  // Continue with the remaining status stream
  for await (const event of gen) {
    switch (event.status) {
      case "SOURCE_FINALIZED":
        // The RefundEscrow POST request has been finalised on the destination chain
        // (the "source" of this ISMP message in protocol terms).
        console.log("RefundEscrow POST finalised:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_DELIVERED":
        console.log("RefundEscrow message delivered to Hyperbridge:", event.metadata.transactionHash)
        break

      case "HYPERBRIDGE_FINALIZED":
        console.log("RefundEscrow message finalised on Hyperbridge:", event.metadata.transactionHash)
        break

      case "CANCELLATION_COMPLETE":
        console.log("Escrowed tokens refunded:", event.transactionHash)
        return
    }
  }
}
```

The SDK encodes the `cancelOrder` calldata for the destination gateway and yields `AWAITING_CANCEL_TRANSACTION`. You sign the transaction with sufficient `value` to cover the Hyperbridge dispatch fee and pass back the raw signed transaction (or its hash). Once confirmed, the generator yields `CANCEL_STARTED` with the transaction hash and block number. The gateway sets `_filled[commitment] = order.user` and emits a `PostRequestEvent` carrying the `RefundEscrow` message. A relayer delivers this message to the source chain via Hyperbridge, where `onAccept` authenticates the sender as the registered destination gateway, decodes the withdrawal request, and issues the refund.

## Quoting the Cancellation Fee

Cross-chain cancellations require a Hyperbridge dispatch fee. The `cancelOrder` generator quotes the fee automatically and includes it as `value` in the `AWAITING_CANCEL_TRANSACTION` event — no separate quote call is needed. `quoteCancelNative` is available if you need the fee estimate upfront (e.g. to display it to the user before they initiate cancellation).

```typescript lineNumbers title="quote-cancel.ts" icon="typescript"
// GET request fee for source-side cancel (for display purposes)
const sourceFee = await intentGateway.quoteCancelNative(order)
console.log("Source cancel fee (wei):", sourceFee)

// POST request fee for dest-side cancel (for display purposes)
const destFee = await intentGateway.quoteCancelNative(order, true)
console.log("Dest cancel fee (wei):", destFee)
```

## Checking Order Status

```typescript lineNumbers title="check-status.ts" icon="typescript"
// Returns true if _filled[commitment] != address(0).
// This is true for both solver-filled orders AND dest-cancelled orders.
const filled = await intentGateway.isOrderFilled(order)

// Returns true if the order was cancelled and escrow was returned (EscrowRefunded event exists).
const refunded = await intentGateway.isOrderRefunded(order)
```

`isOrderFilled` and `isOrderRefunded` can both be true simultaneously — a dest-cancelled order sets `_filled` (so `isOrderFilled` returns `true`) and also triggers a refund (so `isOrderRefunded` returns `true` once the source chain processes `onAccept`). Use `isOrderRefunded` when you specifically need to confirm the user received their tokens back.
